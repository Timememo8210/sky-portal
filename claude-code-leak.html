🔮 技术发现 — 源码里的猛料
🤖 KAIROS — 永不休眠的自主后台代理 未发布 ▸

源码中 150+ 次引用。KAIROS 是一个自主后台守护进程模式，Claude 可以作为持久化、常驻的后台 Agent 运行。

核心特点：接收周期性 <tick> 提示来决定是否主动行动；维护 append-only 日志文件；订阅 GitHub webhooks。

配套功能 autoDream — 后台记忆整合进程，在用户空闲时作为 fork 子 Agent 运行，合并分散观察、消除逻辑矛盾、将模糊洞察转化为明确事实。

🧠 ULTRAPLAN — 远程云容器 30 分钟深度规划 未发布 ▸

将复杂规划任务卸载到远程云容器，运行 Opus 4.6（代号 Fennec），给予最多 30 分钟的专用思考时间。

这是 Anthropic 对"深度思考"能力的产品化尝试 — 不在本地消耗上下文窗口，而是用专门的云实例来处理需要长时间推理的任务。

🐾 BUDDY — 终端宠物 Tamagotchi 系统 趣味彩蛋 ▸

一个完整的 Tamagotchi 风格终端伴侣系统：

🎮 18 种物种，稀有度从普通(60%)到传奇(1%)
✨ 1% Shiny 概率
📊 5 个属性：DEBUGGING、PATIENCE、CHAOS、WISDOM、SNARK

这是 Anthropic 工程师的浪漫 — 在价值 $25亿 ARR 的严肃产品里，藏了一个虚拟宠物。

🕵️ Undercover Mode — 隐藏 AI 身份模式 最具争议 ▸

文件 undercover.ts，约 90 行代码。注入系统提示指令：

⚠️ 永远不提及自己是 AI
⚠️ 移除所有 Co-Authored-By 署名（向外部仓库贡献代码时）

这是本次泄露中争议最大的发现。它意味着 Anthropic 准备了（或至少原型了）让 Claude 在开源项目中匿名贡献代码的能力，这引发了关于 AI 透明度和开源伦理的激烈讨论。

🛡️ Anti-Distillation — 毒化竞争对手训练数据 攻防策略 ▸

ANTI_DISTILLATION_CC feature flag 触发两种防御机制：

🔹 假工具注入 — 向 API 请求中注入虚假工具定义，污染记录 API 流量的竞争对手训练数据
🔹 推理链替换 — 用密码学签名的摘要替代完整推理链，防止对手通过蒸馏复制能力

这揭示了 AI 公司之间隐秘的攻防战 — Anthropic 假设竞争对手正在监听其 API 流量来训练模型。

🏗️ 架构亮点 — 三层记忆 + 40 工具插件 核心架构 ▸

三层记忆架构：

📌 MEMORY.md — 轻量指针索引（~150字符/行），始终加载到上下文
📂 Topic Files — 项目知识按需加载
📄 Raw Transcripts — 从不完整读回，仅 grep 特定标识符

工具系统：~40 个工具插件架构。BashTool 12,400+ 行，四层安全防线。

渲染引擎：React + Ink 终端渲染，游戏引擎式 60fps 技术。46,000 行查询引擎。

多 Agent 编排：Coordinator 模式，一个 Claude 管理多个 worker 并行运行。

💭 结语

这次泄露事件的根源荒诞得令人难以置信 — 一行 .npmignore 配置的缺失，加上一个 20 天无人修复的 Bun bug，就将价值 $25亿 ARR 的产品源码完全暴露。

但真正有趣的不是泄露本身，而是源码里揭示的 Anthropic 的野心：KAIROS 展示了 AI Agent 从"被动工具"到"主动伙伴"的进化方向；ULTRAPLAN 预示了"深度思考即服务"的产品形态；而 Anti-Distillation 和 Undercover Mode 则揭开了 AI 公司之间暗流涌动的攻防战。

代码可以重构，feature flag 可以重命名。但泄露出去的产品战略和架构哲学——三层记忆系统、Coordinator 多 Agent 模式、游戏引擎式终端渲染——这些才是真正无法收回的东西。竞争对手现在可以直接站在 Anthropic 的肩膀上。

对开发者而言，这是一个千载难逢的学习机会。512,000 行生产级代码、40+ 工具的插件架构、工业级 prompt engineering——这些通常只在公司内部可见的技术实践，现在成了最好的开源教材。

教训很简单：npm pack --dry-run。发布前，看看你到底在发什么。
